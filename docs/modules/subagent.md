# subagent

Agent invocation infrastructure extracted from `cai_lib/subprocess_utils.py`. Owns both the typed-options Claude Agent SDK execution path and the deprecated `claude -p` argv facade, plus shared cross-cutting concerns: cost-row attribution, stderr capture, FSM state stamping for telemetry, and SDK error diagnostics.

## Entry points

- [`cai_lib/subagent/__init__.py`](../../cai_lib/subagent/__init__.py) — Base classes `SubAgent` and `CostTracker` (repo-agnostic, zero cai-specific logic).
- [`cai_lib/subagent/core.py`](../../cai_lib/subagent/core.py) — `SubAgent` class drives agent execution via the Claude Agent SDK with typed options; `SubAgent.run(prompt)` returns a typed `RunResult` (replacing the legacy `subprocess.CompletedProcess` shape) so callers can inspect structured outcomes without re-parsing opaque stdout/stderr strings (issue #1277). Owns the `_collect_results` query driver.
- [`cai_lib/subagent/transcript.py`](../../cai_lib/subagent/transcript.py) — Typed Pydantic models for SDK transcripts: `RunTranscript` (top-level container capturing the full event stream as a recursive tree), `SubAgentNode` (nested Task spawns with recursive events), and event types (`AssistantTextEvent`, `ToolUseEvent`, `ToolResultEvent`). The `RunTranscript` exposes derived properties (`parent_model`, `last_assistant_text`, `subagent_counts`) for backward compatibility (issue #1279).
- [`cai_lib/subagent/cost_tracker.py`](../../cai_lib/subagent/cost_tracker.py) — Pydantic-modeled `CostRow` and `ModelUsage` data classes that define the cost-log row schema explicitly with field descriptions (issue #1275).
- [`cai_lib/cai_subagent.py`](../../cai_lib/cai_subagent.py) — Cai-specific subclasses: `CaiCostTracker` (stamps FSM state + caller row extras and posts cost-attribution GH comments), `CaiSubAgent` (pins the npm-installed `claude` CLI and auto-injects the `cai-skills` plugin), and `run_subagent(...)` (one-shot invocation over `CaiSubAgent` with cost tracking and GH-comment integration).
- [`cai_lib/claude_argv.py`](../../cai_lib/claude_argv.py) — Deprecated `_run_claude_p(...)` argv facade — a thin delegating shim (#1274) that parses `claude -p` command-line syntax into `ClaudeAgentOptions`, constructs a `CaiSubAgent` backed by `CaiCostTracker`, and adapts the `RunResult` back to `subprocess.CompletedProcess` for compatibility with existing handlers not yet ported to `run_subagent`.
- [`cai_lib/subagent/fsm_state.py`](../../cai_lib/subagent/fsm_state.py) — Compatibility shim re-exporting `set_current_fsm_state` from `cai_lib.fsm_state` (issue #1269); the actual implementation is in the top-level `fsm_state` module.
- [`cai_lib/subagent/stderr_sink.py`](../../cai_lib/subagent/stderr_sink.py) — Stderr-capture sink wired into `ClaudeAgentOptions.stderr`; bounds the capture and routes SDK errors to logs so transient failures (network, OOM, signal) don't disappear into the wrapper's stream.
- [`cai_lib/subagent/errors.py`](../../cai_lib/subagent/errors.py) — SDK-error diagnostic summariser; parses `ClaudeAgentError` exceptions and emits readable error descriptions for logging and debugging.

## Dependencies

- **config** — imports constants (`REPO`, `CAI_COST_COMMENT_RE`) and logging infrastructure (`log_run`, `log_cost`).
- **github-glue** — calls `_strip_cost_comments` to remove stale cost-attribution comments before re-invoking an agent.
- **audit** (indirect) — cost rows published by this module feed into `cai_lib/audit/cost.py` for reporting and analysis.

## Operational notes

- **SDK stream is now a typed recursive transcript.** The `_collect_results` function builds a `RunTranscript` tree that captures top-level events and recursively nests subagent events under `SubAgentNode` containers (issue #1279). The `RunResult` returned by `SubAgent.run()` and `run_subagent()` carries the full typed `transcript: RunTranscript | None` so callers can traverse the entire event stream without re-parsing opaque `stdout` / `stderr` strings. The `RunTranscript` exposes backward-compatible derived properties (`parent_model`, `last_assistant_text`, `subagent_counts`) so existing call sites continue to work unchanged.
- **Two call paths coexist.** `run_subagent` is the new SDK-native path (issue #1226); `_run_claude_p` is the deprecated argv facade that delegates to `run_subagent` (issue #1274). Both emit identical cost-row schemas so telemetry readers need not distinguish. Migration is in progress (12+ handlers still on `_run_claude_p`); **do NOT delete `cai_lib/claude_argv.py` until all handlers are ported.** `_run_claude_p` is a thin delegating shim — it parses argv into options, constructs a `CaiSubAgent` backed by `CaiCostTracker`, and adapts the `RunResult` back to `CompletedProcess` for backward compatibility.
- **Cost-row schema is typed.** The `CostRow` pydantic model in `cost_tracker.py` declares the full schema explicitly with field descriptions and validation (issue #1275). Both call paths construct `CostRow` instances via `from_result_message()` factory, serialize to JSON via `model_dump(exclude_none=True)`, and ship to `log_cost()` in the config module. Readers of `cai-cost.jsonl` (audit, cost-optimize, cost-report CLI) depend on the canonical schema — any field added must be added to the pydantic model first.
- **FSM state is context-only.** The dispatcher (not the handler) calls `set_current_fsm_state(state.name)` before invoking a handler. Non-FSM call sites (rescue, unblock, audit runners, `cmd_misc.init`) leave the contextvar unset and cost rows simply omit the `fsm_state` key. This is intentional — avoid threading state through 27+ call sites just to add context to a few.
- **Stderr sink is best-effort.** The sink bounds capture size and routes SDK errors to logs; a very large stderr output may be truncated. This preserves observability for crash diagnostics without unbounded log growth.
- **Cost comments are strips-on-re-invoke.** When an issue is re-invoked (e.g. on a retry or re-dispatch), `_strip_cost_comments` removes prior cost-attribution comments before the new invocation posts an updated comment. Stale cost comments are never orphaned.
