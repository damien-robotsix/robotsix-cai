# docs

User-facing prose documentation (rendered via Jekyll on GitHub
Pages), project README, license, shared `CLAUDE.md` subagent
guidance, and the auto-generated file-level index. Also hosts
`docs/modules.yaml` — the module index owned by the
`cai-review-docs` pipeline stage — and the per-module narratives
under `docs/modules/`.

## Key entry points
- [`README.md`](../../README.md) — project overview and usage
  guide.
- [`CLAUDE.md`](../../CLAUDE.md) — shared efficiency guidance
  loaded by every subagent in headless mode (Grep-before-Read,
  work-directory rules, `.cai-staging/` conventions).
- [`CODEBASE_INDEX.md`](../../CODEBASE_INDEX.md) — auto-generated
  file-level index. Regenerated by `scripts/generate-index.sh`.
- [`LICENSE`](../../LICENSE) — MIT license text.
- [`docs/index.md`](../index.md) — documentation site landing page.
- [`docs/architecture.md`](../architecture.md),
  [`docs/cli.md`](../cli.md),
  [`docs/configuration.md`](../configuration.md),
  [`docs/agents.md`](../agents.md),
  [`docs/fsm.md`](../fsm.md) — prose docs. `fsm.md` is
  auto-generated by `scripts/generate-fsm-docs.py`.
- [`docs/_config.yml`](../_config.yml) — Jekyll configuration for
  GitHub Pages.
- [`docs/modules.yaml`](../modules.yaml) — module registry schema.
  Owned by `cai-review-docs`.
- [`docs/modules/*.md`](../modules/) — per-module narratives
  including this file.

## Inter-module dependencies
- Consumed by **every subagent** — `CLAUDE.md` is loaded by
  claude-code headless mode on every invocation.
- Written by **scripts** — `generate-index.sh` produces
  `CODEBASE_INDEX.md`; `generate-fsm-docs.py` produces
  `docs/fsm.md`.
- Consumed by **audit** — `cai_lib/audit/modules.py` parses
  `docs/modules.yaml`; `scripts/check-modules-coverage.py`
  enforces coverage.
- Owned by **agents-review** — `cai-review-docs` keeps `docs/**`,
  `docs/modules.yaml`, and `docs/modules/<name>.md` in sync on
  every PR that adds/renames/deletes tracked files.
- Touched by **workflows** — `regenerate-docs.yml` auto-commits
  drift in `CODEBASE_INDEX.md` and `docs/fsm.md`.

## Operational notes
- **Write protection.** `CLAUDE.md` files are hardcoded as
  sensitive by claude-code headless mode; edits must go through
  the `.cai-staging/claudemd/` staging directory (see root
  `CLAUDE.md` for the exact convention).
- **Auto-generated invariants.** `CODEBASE_INDEX.md` must not be
  edited by hand (description comments live in the generator
  script); `docs/fsm.md` is regenerated from transition tables.
  Both drifts are auto-committed by the regenerate-docs
  workflow, but PRs should regenerate locally to avoid bounce-back
  commits.
- **Module registry contract.** Adding, renaming, or deleting a
  tracked source file requires a matched update to
  `docs/modules.yaml` AND the relevant `docs/modules/<name>.md`;
  `scripts/check-modules-coverage.py` is the gate.
- **Cost sensitivity.** Zero for the files themselves;
  `CLAUDE.md` size directly inflates every subagent token count —
  keep it terse.
- **CI implications.** `regenerate-docs.yml` + `docs-coverage`
  script enforce drift; Jekyll builds run on GitHub Pages and
  fail softly (rendering is best-effort).
